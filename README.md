# 📊 Kanban Board Analytics Module (Agile BI)

Аналитический микросервис для мониторинга и оценки эффективности процессов разработки (Agile/Kanban KPIs). Модуль подключается к базе данных PostgreSQL основного приложения, агрегирует сырые исторические данные по задачам и визуализирует ключевые продуктовые и командные метрики на интерактивном дашборде.

---

## 🎯 Бизнес-ценность проекта
В реальной разработке менеджменту важно понимать: *«Где у команды бутылочные горлышки?»*, *«Сколько времени на самом деле уходит на баги, а сколько на фичи?»* и *«Как приоритет задачи влияет на скорость её закрытия?»*. 

Этот модуль автоматизирует сбор логов из БД и трансформирует их в понятные для бизнеса графики, позволяя оптимизировать SLA и скорость поставки фич (Time-to-Market).

---

## 🛠️ Стек технологий
* **Язык разработки:** Python 3.13
* **Анализ данных и ETL:** Pandas, NumPy
* **База данных:** PostgreSQL (Драйвер: `psycopg2-binary`)
* **Визуализация и BI:** Streamlit, Plotly Express
* **Синтез данных:** Faker (генерация реалистичного 90-дневного лога разработки)

---

## 📈 Какие метрики рассчитываются?

1. **Lead Time (Время выполнения):** Полный жизненный цикл задачи от момента её заведения (создания) в бэклог до перевода в финальный статус `Done`.
2. **Cycle Time (Время в работе):** Чистое время, которое задача провела на этапе активной разработки (в колонке `In Progress`).
3. **Распределение нагрузки (Task Distribution):** Динамика входящего потока задач и соотношение их приоритетов (`Low`, `Medium`, `High`, `Blocker`).
4. **Анализ аномалий (BoxPlot / Ящик с усами):** Выявление задач-зависших, у которых Lead Time сильно превышает средние показатели по их приоритету.

## 🚀 Архитектура и модули приложения

Проект спроектирован по принципу разделения ответственности (Separation of Concerns) и состоит из двух ключевых скриптов:

1. **`db_seeder.py` (Data Engineering / Seeding):**
   * Автоматически проверяет и инициализирует изолированные аналитические таблицы с префиксом `analytics_` в PostgreSQL (не конфликтуя с таблицами основного NestJS API).
   * С помощью библиотеки `Faker` генерирует массив из 300+ реалистичных задач за последние 3 месяца.
   * Имитирует логику Agile-процессов: задачи имеют разное время создания, случайный (но логичный) сдвиг по времени взятия в работу и разную длительность завершения в зависимости от приоритета.

2. **`dashboard.py` (Data Analysis & Visualization):**
   * Подключается к БД, выполняет агрегирующий SQL-запрос с объединением (JOIN) таблиц.
   * На чистом Python (Pandas) рассчитывает дельты времени в днях и часах, фильтрует выбросы.
   * Строит интерактивный веб-интерфейс на `Streamlit` с динамическими фильтрами, где графики `Plotly` мгновенно перестраиваются под выбранные параметры.
